# Bounding Boxes Documentation

## Overview
Geographic bounding boxes are a crucial component of our data filtering pipeline. Because mass shootings are localized events, we aim to isolate Twitter discussions originating specifically from or around the affected communities to gauge localized sentiment and reactions. 

The pipeline filters tweets by checking if their geographic coordinates (`latitude`, `longitude`) fall within a rectangular bounding box encompassing the incident city.

## Collection Methodology
The bounding boxes are collected programmatically using the OpenStreetMap (OSM) **Nominatim API**, accessed via the Python `geopy` library. 

The `src/collect_bboxes.py` script queries the API with the following city names:
1. **Uvalde, TX** (Incident Date: May 24, 2022)
2. **Highland Park, IL** (Incident Date: July 4, 2022)
3. **Monterey Park, CA** (Incident Date: January 21, 2023)
4. **Nashville, TN** (Incident Date: March 27, 2023)

When queried, the API returns a `boundingbox` list structured as:
`[minimum_latitude, maximum_latitude, minimum_longitude, maximum_longitude]`

## Data Storage Format
The collected bounding boxes are stored as a JSON object in `data/bboxes.json` for rapid loading during the multiprocessing pipeline. The format is as follows:

```json
{
    "Uvalde, TX": {
        "lat_min": 29.1738,
        "lat_max": 29.2435,
        "lon_min": -99.8143,
        "lon_max": -99.7426
    },
    "Highland Park, IL": {
        "lat_min": 42.1539,
        "lat_max": 42.2037,
        "lon_min": -87.8384,
        "lon_max": -87.7813
    }
}
```
*(Note: The coordinates above are illustrative examples. Actual coordinates are dynamically generated by `collect_bboxes.py`)*

## How It Works in the Pipeline
During the execution of `src/filter_locations_dates.py`, the pipeline performs the following steps for each tweet:
1. Loads the tweet's `latitude` and `longitude`.
2. Iterates through the predefined list of incident cities.
3. Checks if the tweet's date falls within the $\pm 2$ day window of the incident.
4. If the date matches, it applies a geographic mask:
   ```python
   lat_mask = (df['latitude'] >= bbox['lat_min']) & (df['latitude'] <= bbox['lat_max'])
   lon_mask = (df['longitude'] >= bbox['lon_min']) & (df['longitude'] <= bbox['lon_max'])
   ```
5. Tweets satisfying both temporal and spatial constraints are tagged with their respective `matched_incident` name and retained for keyword filtering.
